# Happy Match 参考视频高保真完善设计

**日期：** 2026-06-27

**状态：** 已获用户口头批准，等待书面规格复核

**目标工程：** Unity 2022.3.62f3c1，`HappyMatchGame`

**主验收环境：** Unity Editor 竖屏 Game 视图；鼠标与移动端触摸输入均保持兼容

## 1. 目标与成功定义

在保留现有 9×8 三消核心玩法的基础上，将 Demo 完善为一段可稳定演示、在构图、节奏、视觉反馈和听觉反馈上高度贴近 43.56 秒参考视频的完整单关卡体验。

成功必须同时满足以下条件：

- 棋盘、目标栏、冬夜背景、雨雪环境和胜利演出的整体构图与参考视频一致。
- 四类普通棋子、橙色行李箱、火箭、炸弹和螺旋桨均具有清晰且统一的材质风格。
- 交换、无效回退、消除、下落、补充、连锁、目标收集、特殊道具和胜负流程均有完整动画与音效。
- 初始 33 个行李箱、目标计数、步数、输入锁和胜负状态保持数据一致，不出现重复计数或动画期间状态错乱。
- 自动化测试、编辑器运行、完整通关、关键截图、音效触发与性能验收全部通过后，才进入用户人工验收。

“高保真复刻”指构图、颜色关系、元素轮廓、交互反馈和演出节奏尽量贴近参考视频；核心资源采用独立绘制或许可明确的免费素材，不直接复制来源不明的商业游戏像素。

## 2. 当前基线与差距

当前工程已有交换、匹配、消除、重力、补充、行李箱、火箭、炸弹、目标 UI 与胜负弹层的基本代码，但仍有明显差距：

- 场景文件本身只有 Main Camera，主要系统依赖运行时临时对象，干净重开后的可运行入口不够可靠。
- 棋子与特殊道具由代码生成简单色块，缺少统一美术资源、材质、粒子和环境层。
- 开局与行李箱位置随机，无法重复复现参考视频的视觉与演示路径。
- 现有 UI 为纯色条和英文文本，与奶油色目标栏及 “Great” 演出不符。
- 尚未实现螺旋桨、行李箱飞向目标栏、目标数字抵达后弹跳、棋盘退场和大行李箱胜利爆发。
- 尚无音频系统、素材许可证清单、EditMode/PlayMode 测试和自动验收场景。

## 3. 范围

### 3.1 必须实现

- 固定、可解、无开局自动匹配的 9×8 单关卡。
- 四类普通元素和 33 个行李箱目标物。
- 相邻点击或拖拽交换、无效交换回退、匹配、消除、下落、补充和连锁。
- 普通匹配清除四邻行李箱；特殊道具直接命中行李箱。
- 横向与纵向火箭、3×3 炸弹、自动追踪行李箱的螺旋桨。
- 火箭＋火箭、火箭＋炸弹和火箭＋螺旋桨三种高价值组合。
- 步数限制、失败重试、目标归零、胜利锁定与完整胜利演出。
- 冬夜背景、持续雨雪、棋盘框、目标栏、元素材质、特殊道具材质、粒子、拖尾和全套音频。
- 测试、验收工具、关键截图与最终验收录像。

### 3.2 不实现

- 多关卡推进、联网、账号、商店、体力、金币、排行榜和存档。
- 完整商业游戏级关卡编辑器。
- 所有可能的特殊道具组合；仅实现本规格列出的三种组合。
- 直接依赖用户个人 Asset Store 登录态的付费或受限资源。

## 4. 视觉设计

### 4.1 构图与分辨率

- 设计参考尺寸为 820×1022，与参考视频 1640×2044 等比。
- 摄像机采用竖屏正交构图，宽高变化时以安全区约束顶部目标栏和棋盘。
- 背景为冷蓝冬夜室内/街景，人物与散落行李位于棋盘后方下部；背景对比度低于棋盘，避免干扰操作。
- 顶部目标栏为奶油色圆角胶囊，中心显示橙色行李箱和剩余数量。
- 棋盘位于画面中下部，使用半透明深蓝底、青蓝描边和轻微内发光；9×8 格子清晰但不画重网格。
- 步数以低干扰小胶囊显示在目标栏侧边；参考演示模式可淡化但不能丢失失败信息。

### 4.2 普通元素

四类元素使用不同轮廓与色相，确保色弱与快速移动时仍可区分：

- 红色：冬帽轮廓，亮红高光、深红底部阴影。
- 蓝色：方形软垫/冰块轮廓，圆角、中心凹点和冷白高光。
- 黄色：手套/保暖帽轮廓，暖黄高光与橙色阴影。
- 绿色：双叶/袜子轮廓，亮绿高光与深绿边缘。

所有棋子共享统一的描边宽度、高光方向、柔和投影和轻微果冻形变。选中时播放 1.06 倍呼吸并显示环形微光。

### 4.3 行李箱与特殊道具

- 行李箱为橙棕色硬壳箱，包含把手、两条竖向绑带、边角高光和受击裂纹/闪白状态。
- 火箭保留基底棋子颜色，叠加高饱和条纹、方向标记、脉冲发光和触发尾焰。
- 炸弹为圆润高亮主体，使用呼吸式边缘发光、短引线火花和爆炸前压缩。
- 螺旋桨为彩色旋翼道具，待机持续旋转；触发后沿弧线飞向剩余行李箱并带白金色拖尾。

### 4.4 环境与粒子

- 前后两层雪粒使用不同速度、尺寸和透明度，叠加少量斜向雨丝。
- 普通消除产生对应颜色碎片、白色星点和短促扩散环。
- 火箭使用方向性光束与尾焰；炸弹使用径向冲击环、火花和短暂屏幕震动；螺旋桨使用弧形拖尾与命中星爆。
- 粒子、飞行目标图标和临时提示均使用对象池，避免连续连锁时产生卡顿。

## 5. 动画与交互时序

### 5.1 基础操作

- 选择反馈：0.10 秒放大并点亮选择环。
- 有效交换：0.16 秒 EaseOut；交换结束后再检测匹配。
- 无效交换：0.14 秒换出、停顿 0.04 秒、0.14 秒回弹，不扣步数并播放低沉回退音。
- 普通消除：0.18 秒，依次包含轻微放大、闪白、碎片和缩小。
- 下落与补充：按移动距离在 0.18–0.34 秒内完成，同列轻微错峰，落地后 0.08 秒回弹。
- 连锁间隔：0.06 秒；每级连锁提升音高、粒子数量和轻微镜头冲击。

### 5.2 行李箱收集

1. 行李箱受击时闪白、压缩并出现裂纹。
2. 逻辑目标立即标记为已清除，防止重复命中。
3. 生成临时行李箱图标，以 0.50–0.65 秒贝塞尔弧线飞向目标栏。
4. 飞行图标抵达后，显示计数递减，目标栏与数字播放 0.16 秒弹跳并触发收集音。
5. 最后一个飞行图标抵达且所有消除队列结束后，才进入胜利演出。

### 5.3 特殊道具

- 火箭：0.12 秒蓄力后在 0.24 秒内扫过整行或整列，命中按距离错峰。
- 炸弹：0.18 秒压缩蓄力后清除中心 3×3，播放径向冲击与短震屏。
- 螺旋桨：0.16 秒起飞，0.45–0.70 秒追踪优先级最高的剩余行李箱；无目标时选择普通元素。
- 特殊道具触发统一进入效果队列；同一格只结算一次，连锁道具按可预测顺序展开。

### 5.4 胜利演出

1. 锁定棋盘输入并等待所有目标飞行动画结束。
2. 目标栏完成态弹跳，棋盘在 0.45 秒内缩小、淡出并向下退场。
3. 背景人物与行李区域提亮，“Great” 文字以 0.24 秒弹性曲线出现。
4. 大行李箱从画面中心放大出现，叠加星芒、金色碎片和短促镜头震动。
5. 播放胜利音效后显示可重试/再次演示入口。

## 6. 音频设计

建立 Master、Ambient、SFX、UI 四个混音分组：

- Ambient：持续寒风、低强度雨雪声，循环点必须无爆音。
- SFX：选择、交换、无效回退、普通消除、连锁层级、行李箱破裂、行李箱收集、火箭、炸弹、螺旋桨飞行与命中。
- UI：目标数字弹跳、按钮、失败和胜利。
- 连锁普通消除复用同一音色但按级别提升音高；高反馈特殊道具对普通消除进行短时压低，避免频率拥挤。
- 同类短音效设置最小触发间隔与最大并发数，连续连锁时不产生刺耳叠音。
- 移动端保留轻震接口；Editor 验收不依赖震动结果。

## 7. 技术结构

### 7.1 配置与数据

- `LevelConfig`：固定初始布局、步数、目标数量、随机种子和演示设置。
- `VisualTheme`：普通元素、行李箱、特殊道具、棋盘、UI、背景和粒子资源引用。
- `AudioCatalog`：全部 AudioClip、音量、音高范围、冷却和并发上限。
- 目标计数分为逻辑剩余数、显示剩余数和待飞行动画数，三者通过明确事件同步。

### 7.2 玩法与表现边界

- 现有棋盘、匹配、重力和特殊道具规则继续承担确定性的逻辑计算。
- `GameFlowController` 负责状态机、步数、胜负和输入门禁。
- `BoardPresenter` 将逻辑结果转换为交换、消除、下落、补充和特殊道具动画序列。
- `TargetPresentation` 负责行李箱受击、飞行、目标栏更新和胜利屏障。
- `VfxPool` 与 `AudioManager` 负责复用资源、并发限制和分组播放。
- 场景中保存稳定的 Bootstrap 入口，避免依赖编辑器中未保存的运行时对象。

### 7.3 状态一致性

- Idle 之外禁止普通输入；结果面板显示后保持 GameOver。
- 无效输入只产生选择取消或无反馈，不改变棋盘与步数。
- 清除集合、行李箱命中集合和特殊道具队列均去重。
- 若稳定棋盘不存在合法交换，则播放洗牌动画并生成无初始匹配且至少有一个合法交换的布局。
- 所有协程退出路径必须恢复或明确转移状态，异常资源缺失时使用内置后备资源并记录清晰错误。

## 8. 素材策略与许可证

采用“定制核心资源＋许可明确的免费素材”混合策略：

- 棋子、行李箱、目标栏、棋盘框和关键特殊道具优先独立制作，确保风格一致。
- 雨雪、火花、爆炸纹理和通用音效可选用 Unity Asset Store 免费资源、Unity 官方示例资源、CC0 或其他允许项目分发的免费素材。
- 每个外部素材必须记录名称、来源 URL、作者、许可证、下载日期和工程内路径。
- 无法确认许可或要求额外账号授权的素材不进入最终工程。
- 最终建立 `Document/ASSET_LICENSES.md`，人工验收前完成核对。

## 9. 测试与验收

### 9.1 EditMode 测试

- 固定布局为 9×8、含 33 个行李箱、无开局匹配并至少存在一个合法交换。
- 横向/纵向 3 连、4 连、T/L 形、重叠匹配与连锁检测正确。
- 无效交换回退且不扣步，有效交换只扣一步。
- 重力与补充后棋盘无空格，目标数量与实际目标状态一致。
- 火箭、炸弹、螺旋桨和三种组合的影响集合正确且无重复计数。
- 死局检测与洗牌生成满足可玩约束。

### 9.2 PlayMode 测试

- 动画期间输入被锁定，结束后恢复 Idle。
- 行李箱显示计数仅在飞行动画抵达时更新，最后一个抵达后才触发胜利。
- 连锁与特殊道具连续触发不会遗留空格、重复对象或卡死状态。
- 步数耗尽显示失败面板；重试后场景完整重置。
- 场景从磁盘干净打开即可运行，不依赖 Hierarchy 中未保存对象。

### 9.3 视觉、音频与性能验收

- 以 820×1022 Game 视图分别截取初始状态、普通消除、行李箱飞行、三种特殊道具和胜利演出关键帧，与参考视频并排检查构图与反馈。
- 完整跑通一局脚本化演示并录制验收视频；视频中不得出现 Console 错误、卡死、错位或计数异常。
- 逐项触发音频目录中的每个事件，检查静音、爆音、错误叠加和音量层级。
- Editor Game 视图以 60 FPS 为目标；高强度连锁期间无明显长帧，活跃临时对象数量保持受控。
- Unity Test Framework 全部通过，Console 无编译错误和运行异常。
