# Demo 开发计划书

# 示例概述

示例视频展示的是一个“消消乐”类手游。背景天气为雨夹雪，背景音响有寒风声，背景为衣衫褴褛的人在行李箱中寻求御寒的衣物；内容主体为棋盘和目标栏，其中棋盘位置位于屏幕中部和下部，目标栏位于上部；玩法为玩家在 9 列 x 8 行棋盘上交换相邻普通元素，使 3 个及以上同类元素连成横向或纵向匹配，触发消除、下落、补充和连锁，并可点击道具施加特殊效果，期间伴随“清脆”、“爆炸”等多种爽感游戏音效，颇具节奏感，关卡目标位于屏幕上部目标栏中，通过移除棋盘中的橙色行李箱通关。

# Demo范围定义

## 基础必须项

- 9x8 棋盘，目标栏。

- 4 类普通可匹配元素：红、蓝、黄、绿，可用占位图形替代。 

- 橙色行李箱目标物，占据棋盘格，初始数量 33。 

- 相邻元素交换输入。 

- 横向 / 纵向 3 连及以上匹配检测。 

- 匹配消除、元素下落、顶部补充、连锁消除。 

- 行李箱目标计数 UI。 

- 行李箱被清除后目标计数减少。 

- 至少一种特殊道具，建议优先实现火箭或炸弹。 

- 胜利判定：目标栏中行李箱目标数归零。 

- Demo 失败判定：使用实现假设补充步数限制或限定时间内未消除全部目标。 

- 基础动画反馈，如交换、消除、下落、目标减少、胜利弹窗等。

## 简化项

- 美术可使用颜色块、简单图标、占位 UI。 

- 背景场景可用静态图或纯色背景，背景音效可去。 

- 行李箱飞向目标栏动画可先简化为缩放 \+ 数字递减。 

- 特殊道具组合可先只实现单一道具触发。 

- 游戏音效可仅保留接口或使用临时音效。 

## 暂不实现项

- 多关卡推进。 

- 关卡编辑器。 

- 商业化道具、体力、金币、商店。 

- 联网、排行榜、账号系统。

- 存档系统。 

- 剧情系统。 

- 复杂特殊道具组合。 

- 完整复刻视频中所有动画曲线和镜头表现。

# 规则整理

## 确定规则

- 棋盘为竖屏三消棋盘，主体为 9 列 x 8 行。 

- 顶部目标栏显示橙色行李箱图标和剩余数量。 

- 初始目标数量为 33。 

- 棋盘内存在橙色行李箱目标物，初始数量与目标栏一致。 

- 普通元素至少包含红、蓝、黄、绿 4 种。 

- 玩家操作后会发生元素交换、消除、下落、补充和连锁。 

- 行李箱被清除后会飞向顶部目标栏，随后目标数字减少。 

- 视频中出现火箭 / 条纹桶、炸弹 / 范围爆炸、螺旋桨飞行器类特殊道具。 

- 目标归零后显示完成状态，并进入 Great 胜利反馈。 

- 视频未展示分数系统、失败画面、多关卡切换。 

- Demo 中玩家通过点击相邻两个格子自动进行交换。 

- 使用游戏音效类型由素材库决定与触发时机为点击特殊道具。

## 假设规则

- 无效交换不会扣步数，并回退到交换前状态。 

- 普通匹配会对四邻相邻的行李箱造成一次清除。 

- 行李箱 Demo 阶段只有 1 点耐久，被命中一次即清除。 

- 特殊道具命中行李箱时直接清除行李箱。 

- Demo 使用步数限制作为失败条件，例如 25 步。 

- 每次有效交换扣除 1 步，连锁不额外扣步。 

## 未知规则

- 视频原玩法是否有固定步数、时间限制或其他失败条件。 

- 特殊道具的准确生成规则和组合规则。 

# Demo模块拆解

## 模块一：关卡与场景系统 

- 模块目标：加载 Demo 关卡，管理棋盘、背景、目标 UI 和结算流程。 

- 主要功能：初始化背景、棋盘、目标栏、步数、胜负状态。 

- 输入与输出：输入关卡配置；输出可操作关卡实例。 

- 依赖模块：棋盘系统、UI 系统、目标系统。 

- 关键：保证关卡初始布局可解，不出现开局死局。 

- Demo 简化方案：只实现一个固定关卡。

## 模块二：棋盘数据系统 

- 模块目标：维护 9 x 8 棋盘格状态。 

- 主要功能：记录格子坐标、元素类型、行李箱、特殊道具、空格状态。 

- 输入与输出：输入格子操作；输出棋盘状态变化。 

- 依赖模块：元素系统、匹配系统、下落系统。 

- 关键点：清除、下落、补充和连锁过程中的状态一致性。 

- Demo 简化方案：不做异形棋盘和不可用格。

## 模块三：玩家输入系统 

- 模块目标：接收玩家对棋盘元素的操作。 

- 主要功能：点击选择、拖拽方向判断、相邻交换、特殊道具点击触发。 

- 输入与输出：输入鼠标 / 触摸事件；输出交换请求或道具触发请求。 

- 依赖模块：棋盘系统、规则系统。 

- 关键点：防止动画过程中重复输入导致状态错乱。 

- Demo 简化方案：动画播放期间锁定输入。

## 模块四：匹配与消除系统 

- 模块目标：识别 3 连及以上匹配并执行消除。 

- 主要功能：横竖扫描、匹配集合合并、消除普通元素、触发行李箱受击。 

- 输入与输出：输入棋盘状态；输出待消除格子和受击目标。 

- 依赖模块：棋盘系统、目标系统、动画系统。 

- 关键点：T / L 形和多组匹配同时发生时的处理顺序。 

- Demo 简化方案：先支持横竖 3 连和 4 连，T / L 以随机概率生成炸弹等特殊道具。

## 模块五：下落与补充系统 

- 模块目标：在消除后恢复棋盘可玩状态。 

- 主要功能：列内重力下落、顶部生成新普通元素、检测自动连锁。 

- 输入与输出：输入空格集合；输出稳定后的棋盘。 

- 依赖模块：棋盘系统、元素生成系统、匹配系统。 

- 关键点：多轮连锁时动画和逻辑同步。 

- Demo 简化方案：按列从下到上结算，不做斜向掉落。 

## 模块六：行李箱目标系统 

- 模块目标：管理关卡目标物和胜利进度。 

- 主要功能：行李箱生成、受击、清除、目标计数递减、胜利检测。 

- 输入与输出：输入命中事件；输出目标数量和完成状态。 

- 依赖模块：棋盘系统、UI 系统、动画系统。 

- 关键点：目标计数与飞行动画之间的时序。 

- Demo 简化方案：P0 可先清除即减数，P1 增加飞向目标栏动画。

## 模块七：特殊道具系统

- 模块目标：复现视频中的高反馈消除手段。 

- 主要功能：火箭清行 / 清列、炸弹范围清除、螺旋桨单点飞行清除。 

- 输入与输出：输入匹配结果或点击触发；输出受影响格子集合。 

- 依赖模块：匹配系统、棋盘系统、目标系统、动画系统。 

- 关键点：特殊道具组合和连锁触发容易引起顺序问题。 

- Demo 简化方案：P0 实现火箭与炸弹单体效果，P1 实现螺旋桨，P2 实现组合。 

## 模块八：UI 与结算系统 

- 模块目标：展示目标、步数、胜利和失败。 

- 主要功能：目标栏数字、目标完成勾选、步数显示、胜利 Great、失败重试。 

- 输入与输出：输入关卡状态；输出 UI 更新。 

- 依赖模块：目标系统、关卡系统。 

- 关键点：UI 动画不能遮挡关键棋盘操作。 

- Demo 简化方案：只做顶部目标栏和结算弹层。 

## 模块九：动画与反馈系统 

- 模块目标：让操作结果可感知。 

- 主要功能：交换动画、消除爆点、下落、道具爆炸、目标飞行、胜利特效。 

- 输入与输出：输入逻辑事件；输出视觉反馈。 

- 依赖模块：全部玩法模块。 

- 关键点：动画完成回调与逻辑状态推进。 

- Demo 简化方案：统一使用短时 Tween，不追求完全复刻视频。

## 模块十：音效反馈系统

- 模块目标：为玩家关键操作和关卡状态变化提供听觉反馈，增强消除、连锁、道具和结算的操作手感。

- 主要功能：播放交换音、无效交换音、普通消除音、连锁消除音、行李箱收集音、火箭 / 炸弹 / 螺旋桨触发音、胜利 / 失败音效。

- 输入与输出：输入玩法事件，例如交换成功、匹配消除、目标清除、特殊道具触发、胜负结算；输出对应音效播放请求。

- 依赖模块：玩家输入系统、匹配与消除系统、行李箱目标系统、特殊道具系统、UI 与结算系统。

- 关键点：多次连锁消除时音效容易过密，需要控制播放频率和音量层级；普通消除音效不能和特殊道具音效混淆。

- Demo 简化方案：先配置 6 类基础音效：交换、无效、普通消除、目标收集、特殊道具、胜利；连锁时复用普通消除音并略微提高音调或音量。

# 开发任务

## 任务 1：基础工程与竖屏场景搭建

- 目标：建立可运行的 Demo 场景。 

- 具体实现内容：设置竖屏分辨率，创建背景层、棋盘层、UI 层、结算层。 

- 实现假设：Demo 以移动端竖屏比例开发，也支持编辑器内鼠标操作。 

- 前置依赖：无。 

- 验收方式：运行 Demo 后能看到背景、顶部目标栏占位、中心棋盘区域占位。

## 任务 2：棋盘数据结构与固定关卡初始化 

- 目标：生成与视频一致的 9 x 8 固定棋盘。 

- 具体实现内容：建立格子坐标、元素类型、行李箱类型，配置初始 33 个行李箱。 

- 实现假设：初始布局固定，不做随机开局。 

- 前置依赖：任务 1。 

- 验收方式：进入关卡后可见 9 x 8 棋盘，顶部目标显示行李箱 33，棋盘上存在 33 个行李箱。

## 任务 3：普通元素生成与渲染 

- 目标：显示 4 类普通三消元素。 

- 具体实现内容：实现红、蓝、黄、绿四类元素的创建、刷新、销毁或隐藏。 

- 实现假设：美术可先使用色块或简单图标。 

- 前置依赖：任务 2。 

- 验收方式：棋盘空位能生成普通元素，玩家能通过颜色 / 图形区分四类元素。 

## 任务 4：相邻交换输入 

- 目标：允许玩家操作棋盘。 

- 具体实现内容：实现点击选择相邻格交换，或拖拽一个元素到相邻格交换。 

- 实现假设：Demo 同时兼容鼠标点击和触摸拖拽中的一种即可。 

- 前置依赖：任务 3。 

- 验收方式：玩家拖动或点击相邻两个普通元素后，两个元素会交换位置；非相邻格不会交换。  

## 任务 5：匹配检测与无效交换回退 

- 目标：实现三消基本规则。 

- 具体实现内容：交换后检测横向 / 纵向 3 个及以上同类元素；无匹配则回退。 

- 实现假设：只有有效交换消耗步数。 

- 前置依赖：任务 4。 

- 验收方式：形成 3 连时对应元素消失；没有形成 3 连时两个元素交换后自动换回。  

## 任务 6：消除、下落、补充与连锁 

- 目标：完成三消循环。 

- 具体实现内容：消除匹配元素，空位上方元素下落，顶部补充新元素，稳定后继续检测连锁。 

- 实现假设：新元素从棋盘顶部生成，按列垂直下落。 

- 前置依赖：任务 5。 

- 验收方式：一次匹配后，空格会被上方元素填补；若下落后再次形成匹配，会自动继续消除。 

## 任务 7：行李箱障碍与目标计数 

- 目标：实现关卡核心目标。 

- 具体实现内容：行李箱占据格子；普通匹配影响四邻行李箱；行李箱被清除后目标数量减少。 

- 实现假设：行李箱 1 点耐久，被相邻匹配或特殊道具命中即清除。 

- 前置依赖：任务 6。 

- 验收方式：在行李箱旁边完成匹配后，相邻行李箱消失，顶部目标数字从 33 递减。 

## 任务 8：火箭特殊道具 

- 目标：实现视频中可见的行 / 列清除类道具。 

- 具体实现内容：4 连生成火箭；点击或交换火箭触发整行或整列清除。 

- 实现假设：横向 4 连生成横向火箭，纵向 4 连生成纵向火箭。 

- 前置依赖：任务 6、任务 7。 

- 验收方式：形成 4 连后生成火箭；触发火箭后对应一整行或一整列元素被清除，命中的行李箱减少目标数。 

## 任务 9：炸弹特殊道具 

- 目标：实现视频中的范围爆炸反馈。 

- 具体实现内容：T / L 形匹配或 5 个以内特殊匹配生成炸弹；触发后清除周围范围。 

- 实现假设：炸弹清除以自身为中心的 3 x 3 范围。 

- 前置依赖：任务 6、任务 7。 

- 验收方式：生成炸弹后点击触发，周围元素消失，范围内行李箱被清除并减少目标数。 

## 任务 10：胜利与失败流程 

- 目标：让 Demo 有完整一局闭环。 

- 具体实现内容：目标数为 0 时进入胜利；步数为 0 且目标未完成时进入失败。 

- 实现假设：视频未展示失败，Demo 暂用 25 步限制。 

- 前置依赖：任务 7。 

- 验收方式：清除最后一个行李箱后，棋盘停止输入并显示 Great；步数耗尽且仍有行李箱时显示失败重试。 

## 任务 11：顶部目标 UI 与步数 UI 

- 目标：清晰展示关卡进度。 

- 具体实现内容：显示目标图标、剩余数量、完成勾选、剩余步数。 

- 实现假设：步数 UI 虽未在视频中出现，但为 Demo 失败闭环补充。 

- 前置依赖：任务 7、任务 10。 

- 验收方式：每清除一个行李箱，目标数字正确减少；目标为 0 后显示完成状态。 

## 任务 12：螺旋桨飞行器 

- 目标：补充视频中明显出现的飞行道具。 

- 具体实现内容：生成螺旋桨道具；触发后飞向一个目标行李箱并清除。 

- 实现假设：优先选择剩余行李箱；无行李箱时随机清除普通元素。 

- 前置依赖：任务 7、任务 10。 

- 验收方式：触发螺旋桨后，道具飞向一个行李箱，该行李箱消失并减少目标数。 

## 任务 13：目标飞行动画与数字弹跳 

- 目标：还原视频中的收集反馈。 

- 具体实现内容：行李箱被清除后生成飞向顶部目标栏的临时图标，抵达后数字递减并播放弹跳。 

- 实现假设：P0 可直接减数，P1 再接入飞行动画。 

- 前置依赖：任务 7、任务 11。 

- 验收方式：行李箱被清除后，玩家能看到小行李箱飞向目标栏，目标数字在抵达时减少。 

## 任务 14：特殊道具组合 

- 目标：提升表现接近视频后段效果。 

- 具体实现内容：支持火箭 \+ 炸弹、火箭 \+ 螺旋桨等组合触发。 

- 实现假设：只实现 1 到 2 种最常见组合，不全量覆盖。 

- 前置依赖：任务 8、任务 9、任务 12。 

- 验收方式：两个特殊道具相邻交换时，产生比单个道具更大范围的清除效果。 

## 任务 15：音效与震动反馈接口 

- 目标：补充操作手感。 

- 具体实现内容：为交换、无效交换、消除、爆炸、目标收集、胜利配置音效触发点。 

- 实现假设：视频音色不作为确定规则，Demo 使用临时通用音效。 

- 前置依赖：任务 5、任务 7、任务 10。 

- 验收方式：触发不同事件时能听到不同音效；关闭音效后玩法不受影响。 

# 验收清单 

- 运行 Demo 后可以看到竖屏背景、顶部目标栏和 9 x 8 棋盘。 

- 顶部目标栏初始显示橙色行李箱和数量 33。 

- 棋盘初始布局中存在 33 个行李箱目标物。 

- 玩家可以通过拖拽或点击相邻元素完成交换。 

- 当交换形成 3 连时，对应元素会被消除。 

- 当交换没有形成匹配时，元素会自动回退且不扣步。 

- 消除后，上方元素会下落，顶部会补充新元素。 

- 当普通匹配发生在行李箱四邻位置时，行李箱会被清除。 

- 当行李箱被清除时，顶部目标数字会减少。 

- 玩家可以生成并触发至少一种特殊道具。 

- 特殊道具命中行李箱时，行李箱会被计入目标进度。 

- 满足胜利条件后，系统会停止棋盘输入并显示 Great 胜利反馈。 

- 满足失败条件后，系统会显示失败面板和重试入口。 

- 动画播放期间连续点击不会导致棋盘错位或目标重复计数。 

- 异常操作，例如点击空格、点击非相邻格、重复点击同一格，不会导致报错或棋盘状态损坏。
