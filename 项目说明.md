# Happy Match Game 项目说明

## 1. 工程环境

- Unity：2022.3.62f3c1
- 主场景：`Assets/Scenes/SampleScene.unity`
- 已验证平台：macOS Standalone Player（Apple M4 / Metal）
- 设计分辨率：820×1022 竖屏

## 2. 运行方式

### Unity Editor

1. 使用 Unity Hub 以 Unity 2022.3.62f3c1 打开本工程。
2. 打开 `Assets/Scenes/SampleScene.unity`。
3. 点击 Play。若误打开空场景，工程内场景保护会自动恢复可玩场景。

### 构建 macOS Player

可在 Unity 的 Build Settings 中选择 macOS 并构建，也可在工程根目录执行：

```bash
/Applications/Unity/Hub/Editor/2022.3.62f3c1/Unity.app/Contents/MacOS/Unity \
  -batchmode -nographics -projectPath "$PWD" \
  -executeMethod HappyMatchAcceptanceBuild.BuildMac \
  -happyMatchBuildPath "$PWD/Build/HappyMatchGame.app" \
  -quit -logFile "$PWD/Build/build.log"
```

## 3. 操作方式

- 点击两个相邻普通棋子完成交换；无效交换会回弹且不扣步。
- 横向或纵向三个及以上同类棋子触发消除、下落、补充和连锁。
- 普通匹配会清除四邻行李箱；点击火箭、炸弹或螺旋桨可直接触发特殊效果。
- 顶部显示剩余行李箱和步数；33 个行李箱清零后进入 `Great` 胜利演出。
- 步数耗尽且目标未完成时显示失败与重试入口。

## 4. 已完成内容

- 9×8 固定、可玩的竖屏三消棋盘。
- 四类普通棋子、33 个行李箱目标物、目标计数与 25 步限制。
- 相邻交换、无效回退、横竖匹配、消除、重力、补充和自动连锁。
- 火箭、3×3 炸弹、自动追踪螺旋桨，以及三类特殊组合逻辑。
- 目标飞向顶部、计数弹跳、输入锁、胜负判断和重试/重播。
- 冬夜环境、前后景雨雪、程序化音效、消除/道具/胜利粒子与镜头反馈。
- 指定图片与字体替换：红帽、黄帽、蓝垫、绿叶、橙色行李箱、火箭、炸弹、螺旋桨、目标栏、Passion One 与 Poetsen One。
- 指定行李箱用于目标图标和胜利主物件，不再使用旧礼物图标或木质宝箱作为当前表现。

完整资源映射见 `Document/ASSET_LICENSES.md`。

## 6. 占位资源与已知问题

- 指定图片目录没有与参考视频中“寒冷室内人物/行李场景”等价的完整背景图，因此当前继续使用项目已有的原创冬夜圣诞背景作为占位；雨雪氛围、暗部层级和竖屏构图保持与参考视频方向一致。
- 指定目录没有音效文件，当前使用运行时程序化合成音效；音效事件齐全，但音色不是参考视频原声。
- 为保留明确的失败条件，顶部额外显示步数徽章；参考视频主要突出目标栏。
- 仅在 macOS Player 完成构建与运行验收；鼠标与触摸输入代码共用，但本轮未在 iOS/Android 真机验证。
- 用户提供素材未附独立商业分发许可证；用于公开商业发布前需再次确认授权范围。
- 原始 `图片/` 和 `font/` 目录保留在工程根目录且不直接参与 Unity 导入；实际运行只依赖 `Assets/Resources/HappyMatch/` 中的精选镜像，因此源码工程可独立编译。
